# 📄 페이지(Page) 엔진 명세서

> 본 문서는 시스템의 고정 컨텐츠 관리를 위한 **독립형 페이지 앱 엔진**의 최종 사양을 정의합니다. 수차례의 리팩토링을 통해 데이터 정합성과 보안성을 확보했습니다.

---

## 1. 개요 (Overview)
- **목적**: 게시판(Board) 데이터와 분리된 고정형 문서(약관, 소개 등) 전용 관리 시스템.
- **아키텍처**: 인스턴스형 앱 구조를 채택하여, 슬러그(Slug) 기반의 개별 페이지를 무한 생성 가능.

---

## 2. 데이터 모델 (Data Schema)

### 2.1 `page` 테이블 핵심 사양
- **`slug`**: URL 고유 식별자 (예: `v1/page/about`).
- **`content_json`**: TipTap 에디터 데이터 (JSONB).
- **`is_active`**: 시스템 전체 노출 제어 (True/False).
- **`is_deleted`**: Soft Delete 정책 적용 (데이터 보존).
- **`published_at` / `expired_at`**: 게시 기간 정밀 제어.
- **`redirect_url`**: 만료되거나 비활성 시 자동 이동 주소.

---

## 3. UI/UX 구현 표준

### 3.1 관리자 인터페이스 (Admin UI)
- **중앙 통합 폼**: 에디터 상단에 제목, 슬러그, 날짜, 활성화 토글을 배치하여 작업 직관성 극대화.
- **강제 초기화**: `{#key editingPage.id}` 블록을 사용하여 수정 클릭 시 에디터 메모리를 완전히 갱신함.
- **데이터 복제**: `JSON.parse(JSON.stringify)`를 통해 편집 중 원본 데이터 오염 방지.

### 3.2 사용자 화면 (User Engine)
- **미니멀리즘 디자인**: 본문 컨텐츠에 100% 몰입할 수 있도록 제목과 업데이트 정보를 하단 footer 영역에 은은하게 배치.
- **실시간 리다이렉트**: 비활성 상태 접근 시 즉시 `redirect_url`로 튕겨내거나 에러 메시지 출력.

---

## 4. 내비게이션 정책 (Navigation)
- **SPA 전환**: 모든 페이지 링크는 리로드(`data-sveltekit-reload`) 없이 부드럽게 전환됨.
- **라우트**: `/v1/page/[slug]` (사용자용), `/v1/admin/page` (관리자용).

---

## 5. 학습 및 성장 (Lessons Learned)
- **반응성(Reactivity)**: Svelte 5에서 복잡한 JSON 데이터를 다룰 때는 반드시 객체 참조를 끊고(Clone) 강제 재생성(Key) 기법을 활용해야 함.
- **정합성(Consistency)**: 목록 조회 API 응답 스키마와 상세 수정용 데이터 구조가 일치해야 수정 모드 진입 시 데이터 유실이 없음.
