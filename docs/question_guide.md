# Chapter 1. 자유로운 소통의 시작: 질문 수정 및 삭제 가이드

이 문서는 웹 애플리케이션에서 가장 기본적이면서 중요한 **질문 수정**과 **삭제** 기능이 어떻게 구현되었는지 설명합니다.

---

## 1. 주요 개념 설명

### 🔐 권한 검사 (Authorization)
내가 쓴 글은 나만 수정하고 삭제할 수 있어야 합니다. 
- **백엔드**: API 요청이 들어올 때 현재 로그인한 사용자와 글쓴이가 일치하는지 항상 확인합니다.
- **프론트엔드**: 작성자가 아닌 경우 아예 '수정'이나 '삭제' 버튼이 보이지 않게 처리합니다.

### 🗑️ 소프트 삭제 (Soft Delete)
데이터를 DB에서 완전히 지우는 것이 아니라, "삭제됨"이라는 표시만 해두는 방식입니다.
- **왜 하나요?**: 실수로 지운 데이터를 복구하거나, 나중에 통계 데이터를 분석할 때 유용하기 때문입니다.
- **방법**: `is_deleted`라는 이름의 스위치(True/False)와 `delete_date`(삭제된 시간) 컬럼을 사용합니다.

---

## 2. 전체적인 작동 흐름

### 수정하기 (Update)
1. **입력**: 사용자가 수정 페이지에서 제목이나 내용을 고칩니다.
2. **검증**: 백엔드에서 글쓴이가 맞는지 확인합니다.
3. **반영**: 성공하면 DB의 `modify_date`를 현재 시간으로 업데이트합니다.

### 삭제하기 (Delete)
1. **요청**: 상세 페이지에서 '삭제' 버튼을 누르면 "정말 삭제할까요?"라고 물어봅니다.
2. **처리**: 백엔드에서 `is_deleted`를 `True`로 바꿉니다.
3. **결과**: 이후 목록을 불러올 때는 `is_deleted`가 `False`인 것만 골라서 가져오기 때문에 사용자 눈에는 사라진 것처럼 보입니다.

---

## 3. 관련 파일 안내

- **백엔드**
    - `models.py`: 데이터의 구조(`is_deleted` 등)가 정의되어 있습니다.
    - `question_crud.py`: 데이터베이스에 실제로 접근하여 값을 바꾸는 로직이 들어있습니다.
    - `question_router.py`: 외부(웹 브라우저)에서 온 요청을 받아 권한을 검사하고 CRUD를 호출합니다.

- **프론트엔드**
    - `[question_id]/+page.svelte`: 삭제 버튼 UI와 사용자 확인창 로직이 있습니다.
    - `[question_id]/+page.server.js`: 버튼 클릭 시 백엔드 API를 호출하는 통로 역할을 합니다.

  1. 게시글 관리 및 가시성 향상
   * 공지사항 (Notice): 관리자가 작성한 글을 목록 최상단에 고정하고 배지를
     표시합니다.
   * 비밀글 (Secret Post): 작성자와 관리자만 볼 수 있는 기능입니다. (비밀번호
     방식 또는 권한 방식)
   * 게시글 예약 등록: 원하는 시간에 글이 자동으로 게시되도록 설정합니다.
   * 인기글 (Hot Posts): 조회수나 추천수가 일정 이상인 글을 별도 탭이나 섹션으로
     노출합니다.
   * 다중 카테고리/태그: 자유게시판, Q&A, 정보공유 등으로 분류하거나 #태그를
     통해 검색을 용이하게 합니다.


  2. 소통 및 피드백 기능 (댓글/반응)
   * 대댓글 (Nested Comments): 댓글에 다시 댓글을 다는 계층형 구조입니다.
   * 댓글 고정 (Pinning): 작성자가 가장 도움이 된 댓글을 댓글창 최상단에
     고정합니다.
   * 댓글 추천/비추천: 댓글에도 반응을 남길 수 있게 합니다.
   * 실시간 알림 (Notifications): 내 글에 댓글이 달리거나 추천을 받았을 때
     웹소켓이나 앱 푸시로 알림을 보냅니다.
   * 멘션 (@Mention): 특정 사용자를 지목하여 댓글을 남기는 기능입니다.


  3. 사용자 편의 기능
   * 리치 에디터 (Rich Text Editor): 말씀하신 대로 이미지를 본문에 삽입하고,
     폰트 스타일, 표, 링크 등을 자유롭게 넣는 에디터(CKEditor, Quill 등)
     연동입니다.
   * 임시 저장 (Drafts): 글 작성 중 브라우저가 꺼져도 내용을 복구할 수 있게 자동
     저장하는 기능입니다.
   * 조회수 중복 방지: 동일인이 짧은 시간 내에 여러 번 새로고침해도 조회수가
     하나만 올라가도록 쿠키나 세션으로 제어합니다.
   * 내 활동 모아보기: 내가 쓴 글, 내가 쓴 댓글, 내가 좋아요 한 글을
     마이페이지에서 확인합니다.


  4. 보안 및 운영 관리
   * 신고 시스템 (Report): 부적절한 게시글이나 댓글을 신고하고 관리자가 처리하는
     기능입니다.
   * 자동 스팸 방지 (Captcha): 봇에 의한 무분별한 광고성 글 작성을 방지합니다.
   * 글 수정 내역 (History): 수정 전의 내용을 관리자나 작성자가 확인할 수 있게
     저장합니다.
   * 권한 세분화: 일반유저, 우수유저, 부관리자, 총관리자 등 등급별로 읽기/쓰기
     권한을 차등 부여합니다.


  5. 데이터 최적화
   * 무한 스크롤 (Infinite Scroll): 페이지 번호 클릭 대신 스크롤을 내리면 다음
     글이 자동으로 로딩되는 방식입니다.
   * 전체 텍스트 검색 (Full-text Search): 단순 제목 검색을 넘어 대용량
     데이터에서도 빠르게 내용까지 검색할 수 있는 엔진(Elasticsearch 등)
     도입입니다.
