# 프론트엔드 통신 표준 정의 (Frontend Communication Standard) - v1.0

이 문서는 Svelte 5 프론트엔드와 FastAPI 백엔드 간의 데이터 통신을 일관되게 관리하기 위한 표준 규격을 정의합니다.

---

## 1. 중앙 집중형 통신 (Centralized API)
- **원칙**: 모든 페이지(`.svelte`)에서 `fetch`를 직접 호출하지 않습니다.
- **구조**: `$lib/api.ts` (공통 엔진)와 각 도메인별 API 파일(예: `board.api.ts`)을 통해 통신합니다.
- **장점**: 
  - 백엔드 URL 변경 시 한 곳만 수정하면 전체 반영.
  - 모든 요청에 공통 헤더(Authorization 등) 자동 주입.
  - 전역 에러 핸들링 (서버 다운, 권한 없음 등) 일괄 처리.

---

## 2. API 호출 규격

### 2.1. 기본 엔진 (`api.ts`)
```typescript
// 모든 API 요청의 부모가 되는 공통 함수
export const fastApi = async (method, url, body) => { ... }
```

### 2.2. 도메인 API (`board.api.ts`)
- 게시판과 관련된 통신만 전담하여 코드의 가독성을 높입니다.
```typescript
import { fastApi } from './api';
export const getBoards = () => fastApi('GET', '/admin/boards');
```

---

## 3. 에러 핸들링 표준
- **401 (Unauthorized)**: 로그인 페이지로 리다이렉트.
- **403 (Forbidden)**: "권한이 없습니다" 경고 출력.
- **500 (Server Error)**: "시스템 점검 중" 안내 및 개발자에게 로그 전송.
