# 🎨 Bootstrap 기반 단순 UI 가이드 & 작업 리포트

이 문서는 제주 프로젝트의 **단순하고 직관적인 UI**를 유지하기 위한 Bootstrap 활용법과 레이아웃 수정 내역을 정리한 가이드입니다.

---

## 1. 📂 최근 작업 내용 (레이아웃 최적화)

### ① 메뉴 보존 및 다이어트
- **메뉴 직렬화**: 모든 메뉴(달력, 파일, 결근계 등)를 사용자님의 원래 스타일대로 한눈에 보이게 나열했습니다.
- **반응형 중단점 (lg)**: `navbar-expand-lg` (992px 이상에서 메뉴 노출, 미만에서 햄버거 메뉴)를 적용했습니다.

### ② 레이아웃 푸시(Layout Push) 방식 도입
- **기존**: 배너가 `fixed`로 떠 있어 메뉴를 가리는 문제 발생.
- **수정**: 배너(Level 2)가 나타날 때 레이아웃 본문을 **Y축으로 밀어내는** 방식을 도입했습니다.
  - **상단 배너**: 네비게이션 바 위에서 메뉴를 아래로 밀어냄.
  - **하단 배너**: 본문 아래에서 본문을 위로 밀어 올림.

## 2. 🚦 Bootstrap 반응형 핵심 원리 (단순 UI용)

### 🧩 중단점(Breakpoints) 활용
Bootstrap은 화면 너비에 따라 클래스가 자동으로 적용됩니다.

| 클래스 점미어 | 화면 너비 (기준) | 용도 |
| :--- | :--- | :--- |
| `sm` | ≥ 576px | 작은 스마트폰 |
| `md` | ≥ 768px | 태블릿 |
| `lg` | **≥ 992px** | **일반적인 햄버거 전환 기준** |
| `xl` | ≥ 1200px | 큰 모니터 |

### 🍔 햄버거 메뉴 수동 조정 (Collapse)
햄버거 표시 여부는 `<nav>` 태그의 `navbar-expand-lg` 등의 클래스가 결정하며, 실제 동작은 Bootstrap의 JavaScript(`bootstrap.bundle.min.js`)가 담당합니다.

## 3. ⌨️ CSS 미디어 쿼리 핵심 패턴

나만의 단순한 스타일을 입힐 때 사용하는 표준 패턴입니다.

### 📱 모바일/태블릿용 (992px 미만)
```css
@media (max-width: 991.98px) {
    /* 모바일에서는 버튼을 더 크게 만들어 터치를 유도합니다. */
    .nav-link {
        padding: 12px 15px;
    }
}
```

### 💻 PC/대화면용 (992px 이상)
```css
@media (min-width: 992px) {
    /* PC에서는 메뉴 사이 여백을 주어 한 줄에 다 나오게 합니다. */
    .navbar-nav .nav-link {
        padding-left: 10px;
        padding-right: 10px;
    }
}
```

## 4. 💡 단순 UI를 유지하는 팁
1. **메뉴가 너무 많을 땐**: 드롭다운(`dropdown`) 클래스를 검토하세요. 비슷한 성격(예: 설정, 관리)을 하나로 묶으면 단순함이 유지됩니다.
2. **여백의 미**: CSS 유틸리티 클래스인 `p-2`, `m-2`를 적절히 섞어 각 컴포넌트 간의 숨통을 틔워주세요.
3. **직관적인 버튼**: 중요한 기능은 `border`나 별도 배경색을 주어 단순 나열형 텍스트와 분리하는 것이 좋습니다.
