# 📋 게시판 모델 정의서 (Board Model Definition)

본 문서는 프로젝트에서 다루는 게시판 시스템의 형태를 **기본형(Standard Board)**과 **확장형(Extended Board)** 두 가지 모델로 명확히 분류하여, 향후 실무 및 다른 프로젝트에 바로 투입할 수 있는 기준을 정의합니다.

---

## 🥉 1. 기본형 게시판 (Standard Board)

**목적**: 관리자 페이지나 복잡한 설정 없이, 어떤 환경에 떼어 붙이더라도 즉시 독자적으로 구동되는 가볍고 튼튼한 코어(Core) 게시판. 최소한의 유지보수 비용으로 최대의 효율을 내는 것을 목표로 합니다.

**주요 포함 기능 (Core Features)**:
- **기본 CRUD**: 게시글(질문) 및 댓글(답변)의 작성, 읽기, 수정, 삭제
- **사용자 및 권한 제어**: 로그인, 회원가입, JWT 기반 인증 및 본인 작성 글/댓글에 대한 권한 검증
- **독립적 미디어 시스템**: 다중 파일 및 이미지 업로드, 900x900 썸네일 자동 변환, 고아 파일 방지를 위한 `deleted_` 격리 방어구조
- **지능형 검색**: 제목+내용+저자 통합 검색 기반으로, `sessionStorage`를 활용해 조회, 이동 후에도 검색 상태가 그대로 유지되는 사용자 경험(UX) 최적화
- **사용자 반응 시스템**: 실시간 조회수 카운팅, 복합 반응(좋아요/싫어요/그저그래요) 기록
- **UI/UX**: 외부 에디터(WYSIWYG) 의존성 없는 순수 `<textarea>` 기반의 담백하고 가벼운 인터페이스

---

## 🥇 2. 확장형 게시판 (Extended / Admin Board)

**목적**: 카테고리 분류, 리치 텍스트 에디터, 시스템 관리 기능 등 다채로운 콘텐츠 운영과 관리 편의성이 요구되는 실무 상용 서비스(예: 그누보드, 워드프레스 코어 수준)를 위한 확장 모델.

**주요 확장 기능 (Extended Features)**:
- **[기본형 게시판]의 모든 기능 상속**
- **동적 카테고리 분류 (테이블 분리 방식)**
  - 카테고리를 별도 테이블(DB)로 관리하여 운영 중 소스코드 수정 없이 어드민에서 카테고리 추가/수정/사용 여부 제어 가능
  - 검색 엔진과 연동된 카테고리 필터링
- **리치 텍스트 에디터 (Rich Text Editor)**
  - Tiptap 등 플러그인 기반 에디터를 적용하여 일반인도 쉽게 다룰 수 있는 툴바 환경 제공
  - 서버에 업로드된 이미지를 에디터 본문 내 원하는 위치에 즉시 삽입(연동)
- **최상위 관리자(Admin) 권한 및 대시보드 API**
  - 게시물 모니터링 및 강제 삭제(블라인드) 처리 기능
  - 회원 관리 및 시스템 카테고리 편집 권한 부여
- **커뮤니티 고도화 (선택)**
  - 공지사항 최상단 고정 (Pin)
  - 대댓글(Reply) 및 무한 스크롤 형태의 트리형 답변 구조

---

## 🛠️ 개발 진행 방침 (Roadmap Integration)

**현재 프로젝트의 최우선 목표는 기본형 게시판(Standard Board)의 완전한 독립과 완성입니다.** 
현재 시점에서 남은 마지막 조각인 **'내가 작성한 댓글(답변)의 수정 및 삭제 기능'**을 구현하여 기본형 1.0 구축을 확정 지을 예정입니다. 그 이후 기획 의도와 서비스 확장성에 따라 '확장형 게시판' 기능을 선택적으로 도입(Migration)합니다.
