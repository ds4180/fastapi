# 내부 문서 및 파일 관리 표준 정의 (Internal File Management Standard)

이 문서는 건강검진 결과, 근로계약서, 면허증 사본 등 민감한 개인정보가 포함된 내부 문서의 보안 저장 및 접근 제어 표준을 정의합니다.

---

## 1. 파일 저장 및 물리적 격리 (Storage Isolation)

보안 문서와 일반 이미지는 물리적으로 완전히 격리된 경로에 저장합니다.

- **일반 업로드 (Public):** `/uploads/public/` (게시판 이미지 등)
- **내부 문서 (Internal):** `/uploads/internal/` (민감 정보 포함)

### 1.1. 파일 저장 규칙
1. **파일명 암호화:** 원본 파일명 대신 `UUID`를 사용하여 파일의 내용을 추측할 수 없게 합니다.
   - 예: `2024_검진_김영민.pdf` -> `a1b2-c3d4-e5f6-g7h8.pdf`
2. **무결성 검증:** 파일 업로드 시 `SHA-256` 해시값을 생성하여 DB에 저장합니다. (위변조 확인용)
3. **메타데이터 저장:** 원본 파일명, 확장자, 용량, 해시값은 DB의 `file_metadata` 테이블에서 관리합니다.

---

## 2. Nginx 보안 설정: 직접 접근 차단

보안 문서 폴더(`/uploads/internal/`)에 대한 외부 브라우저의 직접 접근을 Nginx 레벨에서 원천 차단합니다.

### 2.1. Nginx `internal` 지시어 활용
브라우저가 URL로 직접 파일을 요청하면 `404` 또는 `403` 에러를 반환하고, 오직 백엔드(FastAPI)의 승인을 받은 요청만 파일을 보낼 수 있도록 설정합니다.

```nginx
# nginx_default.conf 예시 설정

location /internal_files/ {
    # 외부 요청에 대해서는 이 위치를 숨김
    internal; 
    
    # 실제 파일이 저장된 서버 경로
    alias /home/lee/uv-code/test/uploads/internal/; 
    
    # 보안 헤더 추가 (캐시 금지 및 강제 다운로드)
    add_header Cache-Control "no-store, no-cache, must-revalidate";
    add_header Content-Disposition "attachment";
}
```

---

## 3. 백엔드(FastAPI) 다운로드 로직: X-Accel-Redirect

성능과 보안을 모두 잡기 위해 `X-Accel-Redirect` 방식을 사용합니다.

### 3.1. 작동 프로세스
1. **사용자 요청:** 사용자가 `/api/download/{file_id}` 호출.
2. **권한 검증:** FastAPI가 세션을 확인하여 권한(L3 또는 본인)을 체크합니다.
3. **감사 로그 기록:** "누가, 어떤 파일을, 언제" 요청했는지 `audit_log`에 기록합니다.
4. **승인 응답:** FastAPI가 실제 데이터를 보내는 대신, Nginx에게 명령을 내리는 헤더만 보냅니다.
   - 헤더 예: `X-Accel-Redirect: /internal_files/a1b2-c3d4...pdf`
5. **파일 전송:** Nginx가 내부적으로 파일을 읽어 사용자에게 직접 전송합니다. (백엔드 부하 감소)

---

## 4. 권한 및 감사 로그 (Audit Log)

### 4.1. 접근 권한 (RBAC 적용)
- **L1 (실무자):** 파일 존재 여부만 확인 가능.
- **L2 (관리자):** 본인 담당 부서원의 파일에 대해 열람 권한 (사유 입력 필).
- **L3 (최고관리자):** 모든 파일 열람 및 삭제 가능.

### 4.2. 감사 로그 항목
파일 열람 시 다음 항목을 반드시 기록해야 합니다.
- `timestamp`: 요청 시각
- `user_id`: 요청자 ID
- `file_id`: 대상 파일 ID
- `ip_address`: 접속 IP
- `access_reason`: 열람 사유 (L2 이상 수동 열람 시)

---

## 5. 파일 생명주기 관리 (Lifecycle)

1. **퇴사자 처리:** 퇴사 후 법적 보관 기간(예: 3년)이 지나면 스케줄러가 해당 파일을 자동으로 영구 삭제합니다.
2. **백업 정책:** `/uploads/internal/` 경로는 매일 별도의 암호화된 백업 저장소로 동기화되어야 합니다.
