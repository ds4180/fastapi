# 권한 관리 표준 정의 (Authority Management Standard) - v1.2

이 문서는 시스템의 보안 유지와 업무 효율성을 위해 사용자 분류, 권한 부여, 그리고 데이터 접근 범위(Scope)를 관리하는 표준 규칙을 정의합니다.

---

## 1. 사용자 분류 및 권한 레벨 (Rank Level)

본 시스템은 0부터 4까지의 5단계 권한 체계를 따릅니다.

| 권한 레벨 | 명칭 (Role) | 주요 대상 | 비고 |
| :--- | :--- | :--- | :--- |
| **0** | `ROLE_GUEST` | 비로그인 / 일반 방문자 | 기본적인 공개 정보만 열람 가능 |
| **1** | `ROLE_USER` | 로그인 완료 유저 | 일반적인 게시판 이용 가능 |
| **2** | `ROLE_STAFF` | 실무 담당자 | 담당 업무 및 관련 데이터 관리 가능 |
| **3** | `ROLE_MANAGER` | 책임 관리자 | **메뉴 미등록 페이지 접근 가능**, 주요 설정 관리 |
| **4** | `ROLE_ADMIN` | 최고 관리자 | 시스템 전체 설정 및 관리자 페이지 접근 |

---

## 2. 🌟 지능형 통합 보안 아키텍처 (Unified Security) 🌟

이 섹션은 본 프로젝트의 가장 핵심적인 **추상화 설계 원칙**을 다룹니다. (개발자 감동 포인트!)

### 2.1. 메뉴 기반 권한 상속 (Inheritance by Navigation) <u>[핵심]</u>
- **원칙**: 모든 데이터(게시판, 페이지 등)의 보안 등급은 별도로 설정하지 않고, **해당 데이터를 가리키는 '메뉴'의 등급을 그대로 상속받습니다.**
- **동적 역추적**: 백엔드 API는 호출 시점에 DB의 메뉴 테이블을 역조회하여, 해당 경로(URL)에 걸린 `min_rank`를 실시간으로 판별합니다.
- **<u>장점</u>**: 관리자가 상단 메뉴바의 권한 숫자 하나만 고치면, **눈에 보이는 메뉴 숨김 + 주소창 직접 접속 차단**이 동시에 완벽하게 이루어집니다. (데이터와 UI 보안의 완전한 일치)

### 2.2. 고립 문서 보호 정책 (Default Closed)
- 메뉴판에 등록되지 않은 '고립된' 주소는 보안을 위해 **기본적으로 Rank 3(책임자) 이상만 접근**할 수 있도록 자동으로 잠깁니다.
- 이는 공사 중인 페이지나 비공개 문서가 실수로 노출되는 것을 원천 차단하는 **지능형 방어 기제**입니다.

---

## 3. 시스템 전역 설정 (System Configuration)

시스템의 성격과 동작은 `system_config` 테이블의 JSONB 데이터를 통해 제어됩니다.

- **홈페이지 자동 리다이렉트**: `homepage` 키에 설정된 URL로 루트 접속 시 자동 이동합니다.
- **지능형 랜딩 페이지**: 홈페이지 설정이 `null`인 경우, 시스템은 자동으로 **전체 메뉴 트리(하위 문서 포함)를 시각화한 사이트 맵**을 대문으로 출력하여 사용자의 길 찾기를 돕습니다.
- **다크 모드 통합**: `theme_mode` 설정을 통해 서버 단에서부터 사이트 전체의 테마(Light/Dark)를 결정하여 클라이언트의 시각적 일관성을 보장합니다.

---

## 4. 기술적 구현 가이드

### 4.1. 백엔드 지능형 체커 (`RankChecker`)
- API 정의 시 `Depends(check_rank(required_rank=X))`를 사용하여 선언적으로 보안을 적용합니다.
- `get_current_user_optional`을 통해 비로그인 사용자에게도 에러 없이 최적화된 데이터를 제공합니다.

### 4.2. 프론트엔드 보안 (Server-side Guard)
- 모든 관리자 페이지(`v1/admin/*`)는 서버 사이드 `load` 함수에서 1차 차단을 수행합니다.
- **주의**: SvelteKit의 `redirect` 명령이 `try...catch`에 막혀 무한 루프를 생성하지 않도록 예외 처리를 엄격히 관리해야 합니다.
