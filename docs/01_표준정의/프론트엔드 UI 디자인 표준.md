# 🎨 프론트엔드 UI 디자인 표준: 클린 카드와 반응형 그리드 시스템

> 이 문서는 시스템의 프론트엔드 UI/UX에 일관성을 부여하고, 모든 디바이스에서 최적의 사용성을 제공하기 위한 디자인 표준 및 반응형 구현 가이드라인을 정의합니다.

---

## 1. 핵심 원칙 (Core Principles)

*   **모바일 우선 (Mobile-First):** 가장 작은 화면에 맞춰 디자인을 시작하고, `sm:`, `md:`, `lg:`와 같은 Tailwind CSS 반응형 접두사를 사용하여 점진적으로 큰 화면에 최적화합니다.
*   **일관된 시각 언어:** 모든 UI 요소는 통일된 둥근 모서리, 그림자, 색상 팔레트, 타이포그래피를 사용합니다.
*   **직관적인 정보 계층:** 복잡한 정보도 카드 기반의 명확한 구조를 통해 쉽게 인지되도록 하며, 작은 화면에서도 가독성을 최우선으로 합니다.

---

## 2. 레이아웃 및 반응형 그리드 (Layout & Responsive Grid)

*   **메인 컨테이너:** `max-w-7xl mx-auto px-4`와 같은 클래스를 사용하여 중앙 정렬 및 가로 최대 너비를 제한하고, 화면 크기에 따라 좌우 패딩을 조정합니다. (예: `px-4 md:px-6 lg:px-8`)
*   **요소 간 간격:** `space-y-` 및 `gap-` 클래스를 반응형으로 정의하여 화면 크기에 따라 요소 간 간격을 유연하게 조절합니다. (예: `space-y-6 md:space-y-10`, `gap-4 md:gap-6`)
*   **그리드 배치:** `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3`와 같이 화면 크기에 따라 컬럼 수를 자동으로 조정하여 요소들이 자연스럽게 재배치되도록 합니다.

---

## 3. 컴포넌트 디자인 (Component Design)

### 3.1 카드 (Cards)

*   **기본 구조:** 각 정보 블록(통계 지표, 관리 모듈 등)은 `bg-white`, `border border-slate-200`, `shadow-lg md:shadow-xl`, `rounded-xl md:rounded-2xl` 클래스를 사용하여 카드 형태로 구현합니다.
*   **반응형 둥근 모서리:** `rounded-xl md:rounded-2xl`와 같이 화면 크기에 따라 둥근 정도를 조절하여 작은 화면에서 과하지 않도록 합니다.
*   **그림자 효과:** `shadow-md md:shadow-lg`와 같이 Tailwind 표준 그림자를 사용하여 통일감 있는 깊이감을 부여합니다.

### 3.2 아이콘 (Icons)

*   **Iconify 사용:** 모든 UI 아이콘은 `@iconify/svelte` 라이브러리의 `Icon` 컴포넌트를 사용하여 렌더링합니다.
*   **아이콘 세트:** 일관성을 위해 `mdi` (Material Design Icons) 세트 사용을 권장합니다.
*   **사이즈 및 색상:** 아이콘 크기는 `w-`, `h-` 클래스로, 색상은 `text-` 클래스로 Tailwind를 통해 제어하며, 필요한 경우 반응형으로 조정합니다. (예: `w-4 h-4 md:w-6 md:h-6`)

### 3.3 버튼 (Buttons)

*   **반응형 크기:** `btn-sm md:btn-lg`와 같이 화면 크기에 따라 버튼 크기가 조정되도록 합니다.
*   **둥근 모서리:** `rounded-lg md:rounded-2xl`와 같이 카드 디자인 표준에 맞춰 둥근 모서리를 적용합니다.
*   **그림자:** `shadow-md md:shadow-xl`와 같이 표준화된 그림자 효과를 적용합니다.

### 3.4 반응형 테이블 (Responsive Tables) - 하이브리드 카드/테이블 뷰

*   **하이브리드 뷰:**
    *   **작은 화면 (`sm:` 이하):** 테이블 `<thead>`를 숨기고 (`hidden sm:table-header-group`), `<tbody>`의 각 `<tr>`를 개별적인 카드 형태의 `div`로 변환하여 세로로 쌓이도록 합니다 (`sm:hidden`). 각 `<td>`에 해당하는 정보는 레이블과 함께 `div`나 `span`으로 재구성하여 가독성을 높입니다.
    *   **큰 화면 (`sm:` 이상):** 일반적인 테이블 구조를 유지합니다 (`hidden sm:block`).
*   **테이블 셀:** `py-`, `px-`, `text-`, `font-` 클래스를 반응형으로 조정하여 작은 화면에서도 가독성과 여백을 최적화합니다.

---

## 4. 타이포그래피 (Typography)

*   **반응형 폰트 크기:** `text-xs md:text-sm`, `text-3xl md:text-6xl`과 같이 화면 크기에 따라 폰트 크기를 조정하여 가독성을 유지합니다.
*   **폰트 패밀리:** `font-['Noto_Sans_KR','Outfit']`와 같은 프로젝트 표준 폰트 사용을 권장합니다.

---

## 5. 애니메이션 (Animation)

*   **페이드 인 (`animate-fade-in`):** 페이지 진입 시 `opacity: 0`에서 `opacity: 1`로 부드럽게 나타나는 애니메이션을 표준으로 사용합니다. (`@keyframes fadeIn` 정의 포함)
