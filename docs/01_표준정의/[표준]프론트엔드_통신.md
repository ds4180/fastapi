# 🌐 프론트엔드 통신 표준 정의 (API Communication)

> 이 문서는 "직접 호출 금지, 비서를 통한 대리 호출만 허용"이라는 **중앙 집중형 API 구조**를 통해 Svelte 5와 FastAPI 간의 효율적인 통신 규격을 정의합니다.

---

## 1. 배경 및 목적 (Background & Motivation)
수십 개의 페이지에서 제각각 `fetch()`를 수행하면, 백엔드 서버 주소가 바뀌거나 공통 헤더(JWT 토큰 등)를 추가해야 할 때 수동으로 모든 코드를 고쳐야 하는 재앙이 발생합니다.

본 표준은 **전역 통신 엔진(`api.js`)**을 구축하여, 프론트엔드의 어떤 페이지에서도 일관된 방식으로 데이터를 주고받고 공통 에러를 자동으로 처리하는 데 목적이 있습니다.

## 2. 기존 방식의 문제점 (Analysis of Current State)
- **코드 파편화**: `.svelte` 파일마다 `fetch('http://localhost:8000/api/...')` 코드가 흩어져 있음.
- **인증 토큰 누락**: API 요청 시마다 수동으로 토큰을 헤더에 넣다 보니, 실수로 누락하여 401 에러가 자주 발생함.
- **예외 처리 부재**: 서버 사정에 의한 500 에러나 권한 없음(403) 상황에서 사용자에게 적절한 안내 페이지를 보여주지 못함.

## 3. 해결 방안 및 핵심 로직 (Solution & Key Changes)

### 3.1 중앙 집중형 통신 엔진 (`$lib/api.js`) 🌟
- **핵심**: 모든 통신을 `fastApi()`라는 공통 함수로 단일화합니다.
- **자동 주입**: 로그인 상태라면 모든 요청 헤더에 `Authorization: Bearer [TOKEN]`을 자동으로 삽입하도록 설계했습니다.
- **전역 가드**: 응답 코드가 401(만료된 토큰)인 경우, 서비스 워커나 통신 엔진 단에서 즉시 로그인 페이지로 튕겨내 보안을 유지합니다.

### 3.2 도메인별 API 모듈화 (Layered API)
- 각 페이지는 통신 엔진을 직접 호출하는 대신, 도메인별 매니저(`board.api.js`)를 통합니다.
- 예: `getBoards()` -> 호출자는 내부 로직을 몰라도 데이터만 받으면 됨.

## 4. 상세 구현 내용 (Implementation Details)

### 4.1 통신 엔진 예시 (자바스크립트)
```javascript
// $lib/api.js (모든 통신의 입구)

/**
 * FastAPI 서버와 통신하는 중앙 집중형 함수
 * @param {string} method - HTTP 메서드 (GET, POST 등)
 * @param {string} url - 요청 주소
 * @param {object} [body] - 요청 본문 데이터
 * @returns {Promise<any>} - 서버 응답 데이터
 */
export const fastApi = async (method, url, body) => {
    const access_token = get(tokenState); // 상태 관리 라이브러리 연동
    const headers = {
        'Content-Type': 'application/json',
        ...(access_token && { 'Authorization': `Bearer ${access_token}` })
    };
    
    const response = await fetch(import.meta.env.VITE_API_URL + url, { method, headers, body });
    // 전역 에러 핸들링 로직 포함...
    return response.json();
}
```

### 4.2 도메인 매니저 사용 예시
```javascript
// $lib/api/board.js (게시판 비서)

/**
 * 게시글 상세 정보를 가져오는 함수
 * @param {number} postId - 게시글 ID
 */
export const getPostDetail = (postId) => fastApi('GET', `/api/v1/post/${postId}`);
```

## 5. 성과 및 학습 포인트 (Outcome & Learning)
- **가동성 및 유지보수**: 백엔드 주소 하나만 바꾸면 수천 개의 API 요청이 즉시 정상화됩니다.
- **보안 강화**: 토큰 관리가 중앙화되어 보안 사고 위험이 획기적으로 줄었습니다.
- **학습 포인트**: **"Proxy Pattern(대리자 패턴)"**을 통신 계층에 적용했을 때 생기는 아키텍처적 유연함을 실전에서 경험하고 표준화했습니다.
