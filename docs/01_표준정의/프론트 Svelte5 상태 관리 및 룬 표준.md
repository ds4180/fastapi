# ⚡ 프론트엔드 상태 관리 표준 (State Management)

> 이 문서는 Svelte 5의 **Runes ($state, $derived, $effect)** 문법을 사용하여 자바스크립트 환경에서 복잡한 상태를 관리하는 표준 방식을 정의합니다.

---

## 1. 배경 및 목적 (Background & Motivation)
기본적인 스토어(`writable`) 방식은 상태 추적이 어렵고 로직이 비대해지는 경향이 있습니다. Svelte 5의 Runes는 반응성을 로직 단위로 캡슐화할 수 있는 강력한 기능을 제공합니다.

본 표준은 **반응형 로직의 일관성**을 유지하여, 코드가 커져도 예측 가능한 상태 변화를 보장하는 데 목적이 있습니다.

## 2. 상태 정의 원칙 (The 3 Runes)

### 2.1 기초 상태 ($state)
- **원칙**: 변경 가능한 가장 작은 단위의 데이터만 `$state`로 선언합니다.
- **예시**: `let count = $state(0);`

### 2.2 계산된 상태 ($derived) 🌟
- **원칙**: 다른 상태를 기반으로 계산되는 값(예: 필터링된 목록, 총합)은 반드시 `$derived`를 사용합니다.
- **효과**: 수동으로 업데이트할 필요 없이, 원본이 바뀌면 자동으로 최신화됩니다.

### 2.3 부수 효과 ($effect)
- **원칙**: DOM 조작, 로컬 스토리지 저장, 외부 API 호출 등 "외부 세계"와 상호작용할 때만 사용합니다.
- **금기**: 다른 상태를 업데이트하기 위해 `$effect`를 사용하지 마십시오. (대신 `$derived` 활용)

## 3. 중앙 집중형 상태 (Shared State)
- **패턴**: 전역적으로 공유해야 할 상태(유저 정보, 알림 목록 등)는 `$state`를 내부적으로 가진 **Class** 또는 **Module**로 만들어 `$lib/runes` 파일에 모아 관리합니다.

## 4. 학습 포인트 (Outcome & Learning)
- **가독성 향상**: 반응성 로직이 클래스 내부에 캡슐화되어 컴포넌트 코드가 훨씬 깔끔해졌습니다.
- **학습 포인트**: **"반응성은 선언적(Declarative)이어야 함"**을 이해하고, `$derived`를 통해 복잡한 계산을 자동화하는 기술을 습득했습니다.
