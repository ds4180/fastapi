# 🛡️ 리소스 접근 및 데이터 보안 표준 (Resource Access Security)

> 이 문서는 시스템 내의 개별 데이터(Row 레벨)에 대한 정교한 접근 제어 규격을 정의합니다. 상위 표준인 `[표준]권한_관리`의 Rank 체계를 기반으로 작동합니다.

---

## 1. 배경 및 목적 (Background & Motivation)
메뉴 권한(Rank)이 있더라도, 특정 예민한 데이터(예: 본인의 연봉 정보, 특정 팀의 프로젝트 문서)는 해당 권한을 가진 사람만 보아야 합니다. 

본 표준은 **리소스 단위의 명시적 권한 부여(ACL Lite)**를 통해 데이터 보안의 정밀도를 높이는 데 목적이 있습니다.

## 2. 보안 레벨 및 소유권 (Security Levels)
모든 **관리 리소스(Managed Resource)**는 생성 시 다음 중 하나의 보안 등급을 가집니다.

1.  **PUBLIC**: 모든 로그인 사용자(Rank 1 이상) 접근 가능.
2.  **INTERNAL**: 관련 그룹(팀) 멤버 및 Rank 2 이상 접근 가능.
3.  **CONFIDENTIAL**: 소유자 및 명시적 권한 부여 대상자만 접근 가능.
4.  **RESTRICTED**: 소유자 및 Rank 4(Admin)만 접근 가능.

## 3. 핵심 로직: 명시적 권한 (Explicit Grant) 🌟
- **m2m 기반 전파**: 특정 리소스를 개별 사용자뿐만 아니라 '그룹(Group)' 단위로 한꺼번에 공유할 수 있습니다. 
- **권한 계층**: `READ` (조회), `WRITE` (수정), `ADMIN` (삭제/공유 관리) 권한으로 세분화합니다.

## 4. 구현 규격 (Interface)
데이터 보안을 위해 다음의 매핑 테이블 구조를 활용합니다.
- **`resource_grants`**: (resource_type, resource_id, grantee_id, permission_level)
    - `resource_type`: 'POST', 'DAYOFF' 등 대상 모델 구분.

## 5. 성과 및 학습 포인트 (Outcome & Learning)
- **정교한 권한 제어**: 이제 특정 '문서'뿐만 아니라 시스템 내의 어떤 '데이터'든 유연하게 권한을 쪼개서 관리할 수 있습니다.
- **학습 포인트**: **"전역 권한(Rank)"**과 **"개별 권한(ACL)"**의 위계를 분리함으로써, 관리 효율성과 보안 정밀함을 동시에 달성했습니다.
