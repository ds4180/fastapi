# 🗺️ 플랫폼 통합 주소 체계 규격 (v2.0)

> 본 문서는 개발자 경험(DX)을 최우선으로 하여, URL 구조와 실제 파일 경로를 1:1로 매핑하는 **단일 책임 주소 체계**를 정의한다. 모든 URL은 개발자가 요청을 처리하는 파일의 위치를 즉시 파악할 수 있도록 설계되었다.

---

## 1. 주소 설계 원칙 (Core Principles)

1.  **개발자 경험 최우선 (Developer-Experience First)**: URL의 주 소비자는 개발자이다. URL의 구조는 사용자 편의나 SEO보다 코드의 명확성, 예측 가능성, 유지보수성에 기여해야 한다.
2.  **버전 우선 (Version-First)**: 모든 주소는 API 및 시스템 버전을 나타내는 `/v1`을 최상위 경로로 가진다.
3.  **레이어 기반 구조 (Layer-Driven Architecture)**: 모든 주소는 기능의 성격에 따라 `app`, `page`, `admin`의 3가지 레이어 중 하나에 명시적으로 속한다.
4.  **경로 = 디렉토리 (Path-as-Directory)**: URL 경로는 SvelteKit의 `src/routes` 디렉토리 구조와 1:1로 매핑되어야 한다.

---

## 2. 주소 구조 상세 (Route Anatomy)

### 2.1 일반 구조
모든 URL은 `/{version}/{layer}/{instance}/{record}/{action}`의 계층 구조를 따른다. 뒤따르는 요소는 상황에 따라 생략될 수 있다.
### 2.2 레이어 정의 (Layer Definitions)

| 레이어 | 경로 구조 | 설명 | 관리 페이지 상세 |
| :--- | :--- | :--- |:--- |
| **`app`** | `/v1/app/...` | '앱 구동기' 기반의 동적 기능. | 타입별 입구 자동 결정 |
| **`pages`** | `/v1/pages/...`| 순수 콘텐츠 페이지. (Legacy) | 관리 기능 없음 |
| **`pages`** | `/v1/pages/...`| **정적 콘텐츠 페이지** (파일 시스템 기반). | 파일 시스템 목록 리스트업 및 경로 조회 |

---

## 3. 타입별 라우팅 및 관리 정책 (Routing by app_type)

`AppRegistry`에 등록된 `app_type`에 따라 `/v1/admin/` 하위의 동작이 자동으로 결정된다.

### 3.1 `INSTANCE` 앱 (예: 게시판)
- **목적**: 하나의 엔진으로 여러 개의 독립된 방(인스턴스)을 운영.
- **관리 주소**: `/v1/admin/{app_id}`
- **동작**: 해당 앱의 **인스턴스 목록**(`BoardConfig` 등)을 보여준다. 목록이 없을 경우 "생성된 인스턴스가 없음"을 안내한다.
- **상세 설정**: `/v1/admin/{app_id}/{instance_id}`에서 개별 인스턴스 옵션 관리.

### 3.2 `PAGES` 레이어 (하드코딩 페이지)
- **목적**: 개발자가 직접 코드로 작성한 정적 페이지.
- **관리 주소**: `/v1/admin/pages`
- **동작**: DB 조회 없이 `svelte/src/routes/v1/pages/` 디렉토리를 스캔하여 존재하는 모든 페이지 파일 목록을 리스트업한다. 파일 수정은 코드 수준에서 직접 수행한다.

### 3.3 `STATIC` & `SYSTEM` 앱 (예: 대시보드, 사용자 관리)
- **목적**: 시스템에 단 하나만 존재하며 물리적 폴더/컴포넌트를 가짐.
- **관리 주소**: `/v1/admin/{app_id}`
- **동작 (중요)**: 별도의 목록 단계가 없으므로, 해당 앱의 **전용 관리 컴포넌트로 즉시 진입**하거나 상세 설정 화면으로 **자동 리다이렉트**한다.
- **에러 방지**: 목록이 존재하지 않아 발생하는 빈 화면(Empty State)이나 404 에러를 방지하기 위해 라우팅 가드에서 타입을 선제적으로 체크한다.

---

## 4. 디렉토리 구조 매핑
URL의 레이어는 `svelte/src/routes/v1/` 하위의 디렉토리와 직접 연결된다.

- `/v1/app/*` 요청은 `svelte/src/routes/v1/app/` 내부의 파일이 처리한다.
- `/v1/pages/*` 요청은 `svelte/src/routes/v1/pages/` 내부의 파일이 처리한다.
- `/v1/admin/app/*` 요청은 `svelte/src/routes/v1/admin/app/` 내부의 파일이 처리한다.
