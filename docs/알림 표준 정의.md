# 전역 알림 시스템 표준 정의 (Notification System Standards)

이 문서는 프로젝트 전체에서 공통으로 사용될 알림 및 업무 지시 시스템의 표준 스펙을 정의합니다. 본 시스템은 단순한 UI 요소가 아닌, 사내 업무 이행 보장 및 책임 소지 명확화를 위한 핵심 도구입니다.

## 1. 시스템 목적
* **업무 지시의 확실한 전달**: 밴드(Band) 등 단순 게시판의 한계를 넘어, 중요 공지를 반드시 정독하게 함.
* **책임 소지 방어**: "확인하지 못했다"는 변명을 시스템적으로 차단하여 관리적 과실을 방지.
* **실시간성 확보**: 웹소켓을 통해 관리자가 등록하는 즉시 모든 기사님 화면에 0.1초 만에 공지 노출.

## 2. 알림 등급 및 특종 동작 (Levels)

| 등급 | 명칭 | 형태 | 동작 특징 | 주요 용도 |
| :--- | :--- | :--- | :--- | :--- |
| **Level 1** | **일반 알림** | Toast (팝업) | 10초 후 자동 소멸, 사용자 클릭 방해 없음 | 저장됨, 삭제 완료, 가벼운 안내 |
| **Level 2** | **상태 배너** | Fixed Banner | 해결 전까지 상/하단 고정 (레이아웃 하향 푸시) | 네트워크 불안정, 서버 점검 예고 |
| **Level 3** | **일반 확인** | Modal (중앙) | 사용자의 확인 클릭 전까지 화면 차단 | 삭제 재확인, 규정 변경 인지 |
| **Level 4** | **강제 지시** | Timed Modal | **지정 시간 대기 필수 + 펄스(Pulse) 시각 효과** | 배차 공지, 긴급 안전 지시 |
| **Level 5** | **시스템 락** | Full Screen | 내비게이션 포함 전체 기능 차단 (긴급 점유) | 비인가 유저, 시스템 전면 긴급점검 |

### 🛠 Level 4 "인과응보" 로직
* **목적**: 정독 유도 및 기강 확립.
* **작동**: 확인 버튼에 5~10초의 카운트다운 표시.
* **패널티**: 시간이 다 되기 전에 버튼을 클릭할 경우 **타이머가 최초 설정 시간으로 리셋**되며 얄미운 경고 문구가 출력됨 (예: "지시 사항을 정독하십시오.")

## 3. 실시간 통신 및 네트워크 표준 (Communication)

### ① 웹소켓(WebSocket) 기반 실시간 푸시
* **동작**: 서버의 데이터가 변경되면 웹소켓 통로를 통해 `{"type": "new_alert"}` 신호를 즉시 브로드캐스트함.
* **클라이언트 반응**: 신호를 수신한 브라우저는 즉시 `/api/alert/active` 엔드포인트를 호출하여 알림 목록을 최신화함.
* **안정성**: 웹소켓 연결 유실 시 **지수 백오프(Exponential Backoff)** 재연결 로직 작동 및 **30초 백업 폴링(Polling)** 병행.

### ② 주소 체계 표준 (The Slash Standard)
Nginx 프록시와 백엔드 간의 경로 충돌을 방지하기 위한 절대 규칙입니다.
* **프론트엔드 (Client)**: 모든 요청 주소 앞에 `/api/`를 붙임 (예: `fetch("/api/alert/active")`).
* **백엔드 (FastAPI)**: 모든 라우터 `prefix`에서 `/api`를 제거 (예: `prefix="/alert"`).
* **서버 간 통신 (SSR)**: SvelteKit 서버가 백엔드(8000포트)에 직접 쏠 때는 `/api`를 붙이지 않음.

## 4. 표준 데이터 규격 (Interface)

```javascript
{
  id: "unique_id",          // 중복 방지 및 확인 기록용
  type: "toast" | "modal",   // UI 형태
  style: "info" | "success" | "warning" | "danger", // 시각적 테마
  message: "공지 내용",      // 본문
  level: 1 | 2 | 3 | 4 | 5,  // 작동 등급
  reset_sec: 5,              // Level 4용 대기 시간
  confirm_text: "확인함",     // 버튼 텍스트
  route: "/question-list",   // 노출 경로 (비어있으면 전역)
  redirect_url: "/news/123", // 확인 클릭 시 이동할 주소
  create_date: "ISO Date"    // 생성 일시
}
```

## 5. 구현 가이드라인 (Implementation)

### ① 백엔드: 비동기(Async) 및 복합 전파 처리
* 실시간성 보장을 위해 모든 알림 로직은 `async def` 기반으로 동작함.
* **Level 5 발송 로직**: `await manager.broadcast` (웹소켓) 실행 후, `push_service.send_push_to_all`을 호출하여 오프라인 사용자까지 커버함.

### ② 프론트엔드: Svelte 5 반응성 보장
데이터 수신 시 Svelte 5가 변화를 확실히 인지하도록 새 배열 참조를 생성함.
```javascript
// ✅ 권장되는 스토어 갱신 방식
alerts.set([...json]);
```

### ③ UI/UX 최적화
* **Shimmer 효과**: Level 2 배너에 쉬머 애니메이션을 적용하여 시선 집중 유도.
* **Pulse 효과**: Level 4 모달에 펄스 애니메이션을 추가하여 긴박감 조성.
* **Quiet Mode**: 사용자가 상단바의 🔔 버튼으로 DND 모드를 활성화할 수 있으나, 중요 지시(Level 4 이상)는 이를 무시하고 노출됨.

## 6. 최종 보루 (SOP)
시스템 장애를 대비하여, 중요 업무 안내는 아날로그(종이 출력) 이력 관리와 병행함을 원칙으로 함.
