# 권한 및 보안 구현 로드맵 (Implementation Roadmap)

이 로드맵은 설계된 권한 관리 및 보안 체계를 업무 스트레스를 최소화하며 단계적으로 구현하기 위한 가이드라인입니다. 한 단계씩 확실히 검증하며 진행합니다.

---

## 🏁 Phase 1: 기초 데이터 모델링 (Foundation)
가장 핵심적인 사용자 식별 체계를 잡는 단계입니다.
- [ ] **DB 모델 수정:** `User` 모델 분리 및 `UserProfile` 기본 테이블 생성.
- [ ] **기초 데이터 마이그레이션:** 기존 사용자 정보를 새 구조로 안전하게 이전.
- [ ] **관리자 계정 생성:** 사무실에서 아이디를 부여할 수 있는 최초의 L3(최고관리자) 생성 스크립트 작성.

## 📋 Phase 2: 인사 및 업무 정보 확장 (Profiles)
현장의 방대한 데이터를 담을 그릇을 완성하는 단계입니다.
- [ ] **상세 항목 반영:** 면허, 보험, 차량, 입사일 등 수십 개의 필드를 `UserProfile`에 추가.
- [ ] **동명이인 대응:** `nick_name` 필드를 활용한 UI 노출 로직 구현.
- [ ] **개명 대응:** `real_name` 수정 기능 및 이력 관리(필요시).

## 🔐 Phase 3: 보안 및 내부 문서 관리 (Security)
민감 정보(건강검진 PDF 등)를 보호하기 위한 기술적 장치를 마련합니다.
- [ ] **Nginx 설정:** `internal` 지시어를 활용한 업로드 폴더 접근 차단.
- [ ] **보안 다운로드 API:** `X-Accel-Redirect`를 이용한 권한 기반 파일 전송 로직 구현.
- [ ] **감사 로그(Audit Log):** 파일 열람 이력을 기록할 DB 테이블 및 로직 추가.

## ⏰ Phase 4: 비즈니스 로직 및 자동화 (Automation)
시스템이 스스로 업무를 챙겨주는 단계입니다.
- [ ] **검진/면허 스케줄러:** 만료일이 다가오는 데이터에 대해 자동 알림(Push/Alert) 발송.
- [ ] **배차 시스템 연동:** 자격 미달자(검진 미필 등)의 배차를 자동으로 차단하는 필터 적용.
- [ ] **개인정보 파기:** 퇴사자 데이터의 법적 보관 기간 종료 후 자동 삭제/익명화 처리.

---

## 🛠️ 개발 지침 (Guidelines)
1. **Surgical Update:** 한 번에 다 고치지 말고, Phase 단위로 API를 하나씩 만들어서 테스트합니다.
2. **Safety First:** 주민번호 등 민감 데이터는 가장 나중에, 암호화 로직이 완성된 후 입력합니다.
3. **User Feedback:** 드라이버 한두 명의 실데이터를 넣어보며 사무실 실무자의 피드백을 수시로 반영합니다.
