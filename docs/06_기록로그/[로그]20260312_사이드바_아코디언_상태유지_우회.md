# ✅ 사이드바 아코디언 상태 유지 및 SPA 구조 완성 기록

## 1. 개요 (Context)
- **과거 문제**: `data-sveltekit-reload` 정책으로 인한 전체 페이지 새로고침 시 사이드바의 아코디언 상태(열림/닫힘)가 유실됨.
- **초기 조치**: URL을 분석하여 강제로 아코디언을 여는 `$effect` 로직을 "우회(Workaround)"로 적용함.

## 2. 본질적 해결 (Root Cause Resolution) 🌟
- **원인 분석**: 범용 앱 런처(`app/[app_id]`)가 `const` 변수를 사용하여 앱 전환 시 화면을 갱신하지 못했던 것이 강제 리로드 정책의 근본 원인이었음.
- **최종 해결**: 
    1. 앱 런처에 Svelte 5 **반응성($derived)**을 주입하여 리로드 없이 화면 전환 성공.
    2. 모든 메뉴에서 `data-sveltekit-reload`를 제거하여 **완전한 SPA(Single Page Application)** 환경 구축.
- **결과**: 이제 페이지 이동 시 사이드바 컴포넌트가 파괴되지 않으므로, **아코디언 상태가 메모리에 영구 유지됨.**

## 3. 결론 및 학습 포인트
- **상태 보존**: 더 이상 아코디언 상태 유실을 걱정할 필요가 없음.
- **기능의 승격**: 기존의 우회 로직(URL 기반 자동 확장)은 이제 SPA 환경에서 사용자의 위치를 시각적으로 가이드하는 **정식 편의 기능**으로 승격됨.
- **기술적 교훈**: "작동하지 않는 기능의 배후에는 반드시 정적인 데이터 구조(Reactivity 누락)가 숨어있다"는 점을 명확히 이해함.
