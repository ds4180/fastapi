# 🚀 CMS v1 지능형 게시판 엔진 구축 세션 결과 보고서

**작업 일자**: 2026년 3월 2일 (월)
**핵심 목표**: 메뉴 기반 보안과 연동되는 무한 확장형 게시판(Board) 시스템 구축

---

## 1. 🛡️ 보안 아키텍처 (Intelligent Security)
- **지능형 경로 매핑 (Intelligent Path Mapping)**: 
    - 사용자가 보는 주소(`/board/notice`)와 백엔드 API 주소(`/api/v1/board/list/notice`)가 달라도, 시스템이 자동으로 이를 연결하여 메뉴에 설정된 권한을 상속받도록 구현.
    - **효과**: 관리자가 상단 메뉴바의 권한 숫자 하나만 고치면 주소창 직접 접속까지 실시간으로 차단됨.
- **고립 문서 보호**: 메뉴에 등록되지 않은 모든 경로는 보안을 위해 자동으로 **Rank 3(책임자)** 이상만 접근 가능하도록 설계.

---

## 2. ⚙️ 백엔드 엔진 (FastAPI & SQLAlchemy)
- **이중 저장 전략 (Dual-Storage)**:
    - TipTap 데이터를 `content_json` (편집용)과 `content` (출력/검색용 HTML)로 동시 저장하여 에디터 타입 변경 시에도 데이터가 깨지지 않도록 보장.
- **유연한 데이터 구조**: 
    - `BoardConfig`의 `options`와 `fields_def`를 활용해 코드 수정 없이 게시판의 기능을 정의.
    - `Post`의 `extra_data` (JSONB) 필드를 통해 모든 커스텀 폼 데이터를 안전하게 수용.
- **M2M 고도화**:
    - **PostRead**: 누가, 언제, 몇 번 읽었는지 기록하는 정교한 읽음 확인 시스템.
    - **PostReaction**: 좋아요/싫어요 등 상호작용 기능의 뼈대 구축.
- **DB 무결성 복구**: 
    - 기존 데이터의 NULL 값을 모두 정리하고 `NOT NULL` 제약 조건을 완벽히 복구하여 스키마 안정성 확보.

---

## 3. 🎨 프론트엔드 혁신 (Svelte 5)
- **중앙 집중형 API 시스템**:
    - `api.js` (핵심 엔진)와 `board.api.js` (도메인 비서)로 통신 로직을 추상화하여 유지보수성 극대화.
- **관리자 UI/UX 업레이드**:
    - 메뉴 관리와 동일한 **2단 레이아웃(목록+폼)** 적용.
    - **URL 복사 기능**: 메뉴 등록에 바로 쓸 수 있는 주소를 클립보드에 즉시 복사.
    - **상태 배지**: 게시판의 설정 상태(댓글, 파일, 에디터 타입)를 아이콘으로 요약 노출.
- **지능형 글쓰기 폼**:
    - 게시판 설정에 따라 `TipTap`, `Basic Text`, `Custom Field`, `Read-only` 모드로 화면이 자동 전환.
- **풀옵션 TipTap 에디터**:
    - 표(Table), 텍스트 정렬, 글자 색상/하이라이트, 유튜브 임베드, 체크리스트 등 전문가 수준의 도구 장착.

---

## 📋 커스텀 폼(Field Mode) 지원 타입
관리자 페이지에서 아래 타입들을 조합해 무한한 형태의 폼을 생성할 수 있습니다.
1. **문자열**: 일반 텍스트 입력
2. **숫자(단일)**: 정수/실수 입력
3. **숫자 구간(From~To)**: "5에서 7 사이"와 같은 범위 입력
4. **날짜 / 날짜+시간**: 캘린더 및 시간 피커 연동
5. **범위(Slider)**: 0~100 수치 조절
6. **긴 문장**: 여러 줄 설명 입력
7. **필드 도움말**: 각 항목별 사용자 가이드 문구 지원
