[프론트엔드 복원 상세 내역]

1. 로그인/로그아웃/세션/토큰: 
   - 서버 훅(`hooks.server.js`) 기반 이중 방어 시스템 구축.
   - HttpOnly 쿠키와 로컬스토리지 동기화를 통한 새로고침 시 세션 유지 정책.
   - API 401 에러 대응, 토큰 자동 리프레시 및 경쟁 상태(Race Condition) 방지.

2. 스토어(Store): 
   - Svelte 5 `$state` 룬 문법 기반의 반응형 상태 관리 체계로 전환.
   - 서버 로드(`+layout.server.js`)와 클라이언트 스토어 간 즉시 동기화 로직 적용.

3. API 및 추상화:
   - SvelteKit `handleFetch`를 통한 표준 통신 레이어 추상화.
   - `credentials: 'include'` 기반의 PWA 환경 쿠키 전송 보장.

4. 메뉴 및 권한 관리:
   - 레이아웃 그룹`(app)`, `(auth)`을 통한 메뉴 노출 제어 및 인증 가드 격리.
   - 로그인 여부에 따른 동적 UI 렌더링.

5. 게시판(CRUD) 복원:
   - 등록, 상세 조회, 수정, 삭제 표준 API 호출 로직 재구현.
   - 백엔드 Pydantic 스키마(subject) 준수 바인딩.

   - Redis 기반 동시 접속자 수 관리 및 세션 무효화 정책.
   - 로그인 이벤트 로그 기록(`LoginLog` 모델 연동).

9. 대시보드 및 가입자 관리:

10. 실시간 웹소켓(WS) 및 알림 시스템:
