# 📋 프로젝트 핸드오버 스냅샷 (2026-03-12)

## 1. 핵심 아키텍처 상태 (v1.9.0)
*   **완전한 SPA 전환**: 범용 앱 런처(`app/[app_id]`)의 컴포넌트 호출부에 `$derived` 반응성을 주입하여 `data-sveltekit-reload` 없이도 앱 간 전환이 부드럽게 이루어짐.
*   **페이지 엔진(Page Engine) 완성**: 고정 컨텐츠(약관, 소개 등) 관리를 위한 독립 앱 구축. Tiptap 에디터 및 정교한 게시 옵션(활성화, 기간 설정, 리다이렉트) 완비.
*   **인증 안정성 확보 (v1.9.2)**: `hooks.server.js`에 경쟁 상태(Concurrent Refresh) 방지 로직을 추가하여 빠른 페이지 이동 시 로그아웃되는 현상 해결.
*   **주소 체계 표준 수립**: `/v1/app/...` (사용자 SPA), `/v1/admin/...` (관리자) 중심의 통합 URL 규격 정의 완료.

## 2. 레이아웃 및 UI 상태
*   **3분할 凹자형 레이아웃**: 좌측(탐색), 중앙(실행/에디터), 우측(보조 도구) 구조 정립.
*   **데이터 정합성 해결**: `JSON Clone` + `{#key}` 기법을 통해 Svelte 5 환경에서 에디터 본문 데이터가 유실되던 문제 최종 해결.
*   **디자인**: 사용자용 페이지는 본문 몰입감을 위해 제목과 메타데이터를 하단 footer 영역에 배치.

## 3. 주요 파일 변경 사항
*   `svelte/src/hooks.server.js`: v1.9.2 인증 정합성 및 무한 루프 방지 로직.
*   `svelte/src/lib/components/PageEngine.svelte`: 사용자용 페이지 엔진 본체.
*   `svelte/src/routes/v1/admin/page/+page.svelte`: 페이지 관리자 (Source of Truth).
*   `svelte/src/lib/components/PageAdminEngine.svelte`: 표준 Bridge 컴포넌트.
*   `domain/page/`: 페이지 전용 백엔드 CRUD 및 라우터.
*   `docs/01_표준정의/[표준]플랫폼_주소_체계_규격.md`: 신규 주소 표준 문서.
